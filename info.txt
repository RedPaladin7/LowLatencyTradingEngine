Socket:
- disable nagle -> ip protocol level 
- timestamp -> socket level 
- join multicast ->ip protocol level 
- reuse addr -> socket level
- socket() function takes as params -> family, socket type, protocol

- underlying data structure -> vector<char>, one for inbound data, one for outbound data
- send function loads data into outbound buffer
- sendAndRecv functions receives all data, and sends all queued data 
- callback function called after data is received, only takes the timestamp

Mcast Socket:
- also has inbound_data_ and outbound_data_ vectors 

Server:
- has list of sockets to receive from and send to
- send_sockets -> sockets which are ready to receive data from us 
- recv_sockets -> sockets which have data for us to read
- epoll events array to store any new events returned by epoll wait
